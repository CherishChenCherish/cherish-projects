Gold Monitor

Live — 24/7 via launchd

Overview

A production trading signal system for XAUUSD (gold) that runs autonomously on macOS. Monitors price every 30 seconds, performs multi-factor technical analysis across 3 timeframes, enforces strict risk-reward criteria, and pushes actionable alerts to mobile via Bark.

Built on quantitative analysis experience from Gate.io (RF+RNN ensembles on 50K+ users, 14% engagement lift) and Mercer Consulting (DBSCAN/ARIMA on compensation data). Backtesting methodology informed by Kaggle competition practice (Top 5%, 11,962 entrants).

Architecture

gold-api.com (30s poll) ↓ data_layer.py → OHLCV candles, ATR, volume, structure ↓ signal_engine.py → 5-factor scoring (0–9 pts) │ • Multi-timeframe alignment (1m / 5m / 1h) │ • RSI overbought/oversold confirmation │ • MACD crossover alignment │ • Volume vs. moving average │ • Bollinger Band position ↓ Gate: score ≥ 4.5 AND R:R ≥ 2.0 ↓ monitor_engine.py → Bark push + macOS dialog + voice ↓ gold_web.py → Real-time WebSocket dashboard

Signal System

Multi-Factor Scoring

5 technical factors scored 0–9. Only signals scoring 4.5+ fire. NY session requires 6.0+ (backtested: 15% win rate below).

Risk Management

ATR-based stop losses at 1.5× ATR. Minimum 2:1 risk-reward enforced before any alert fires.

Structural Levels

Dynamic breakout, resistance, recovery, support, and demand zone tracking with periodic auto-update.

Sentiment Overlay

News keyword scoring, economic calendar event suppression, and DXY inverse correlation checks.

Key Decisions

  • Backtested thresholds: MIN_SIGNAL_SCORE=4.5 and ATR_STOP_MULT=1.5 optimized on historical data, not arbitrary
  • NY session gate: Score threshold raised to 6.0 during New York hours after backtesting showed poor win rates below
  • 15-min cooldown: Prevents alert fatigue from repeated signals at the same level
  • Full trade logging: Every signal logged with entry, stop, target, score, and contributing factors in JSONL

Tech Stack

Python, FastAPI, WebSocket, gold-api.com, Bark push, macOS launchd, JSONL logging, sentiment analysis

Live Assistant

Beta — Core complete

Overview

A real-time AI interview coach. Listens to your interview via microphone, automatically detects when the interviewer asks a question, and streams a suggested answer to your screen — all in under 8 seconds from question to first visible token.

Draws on NLP research experience: built a Sentence-BERT + GPT Chrome extension classifying 8K+ comments as a Research Assistant, and integrated toxicity detection across 100K+ LLM interactions (35% toxic content reduction). Pattern-matching and streaming architecture informed by production pipeline work at Alibaba (50-100 GB/day).

Architecture

Microphone (sounddevice, 16kHz, 4s chunks) ↓ faster-whisper (CPU, int8 quantized) ↓ QuestionDetector │ Stage 1: 45+ heuristic patterns (instant) │ Stage 2: LLM verify (weak signals only) ↓ Responder → Ollama HTTP streaming API │ • Behavioral: 3–4 sentence spoken answer │ • Technical: verbal + code + explanation ↓ asyncio.Queue → WebSocket (50ms poll) ↓ Frontend: token-by-token streaming display

Features

Real-Time Transcription

Whisper speech-to-text with VAD filtering. System audio capture via BlackHole for virtual meetings.

Smart Detection

45+ interview patterns with preamble extraction. Strong matches skip LLM for 5s faster response.

Streaming Responses

Token-by-token via Ollama streaming API. Cursor-pulse animation while generating.

Follow-Up Predictions

Predicts likely next questions + generates keyword talking points asynchronously.

Context-Aware

Upload resume, JD, or notes. Responses reference your actual background and experiences.

Meeting Detection

Auto-detects Zoom, Teams, WeMeet, Google Meet and recommends the correct audio device.

Key Decisions

  • Thread-safe WS bridge: Background threads use asyncio.Queue.put_nowait() → async consumer → ws.send()
  • HTTP API over subprocess: Saves 2–3s per Ollama call from process startup overhead, enables streaming
  • 350-char response cap: Must be readable in 5 seconds during a live interview
  • Model warmup on startup: 1-token ping pre-loads model into memory before first real question

Tech Stack

Python, FastAPI, WebSocket, faster-whisper, Ollama (gemma3:4b), sounddevice, HMAC-SHA256 license system

JobPilot

Live — Deployed on Railway + Vercel

Overview

Most job boards are graveyards of ghost postings. JobPilot solves this: upload your resume, and the system searches 50+ curated companies via Greenhouse & Lever APIs, verifies every posting is actually open (not a ghost job), then generates tailored cover letters using Claude — all in one pipeline.

Deployed as a production SaaS with Google OAuth, Stripe subscription billing, and a daily refresh daemon that re-verifies cached jobs. Built to solve a real pain point during the 2026 job market.

Architecture

Resume (PDF) → parser.py (PyMuPDF + Claude extraction) ↓ searcher.py → Greenhouse & Lever APIs (50+ companies) │ Profile-aware query building │ Degree-strict filtering (no PhD-only for MS) │ Category-tagged company matching ↓ verifier.py → Paranoid verification pipeline │ 1. ATS API check (100% definitive) │ 2. HTML signal analysis (closed/open/ambiguous) │ 3. Ghost job detection (age > 60 days = suspect) │ Philosophy: honest "unverified" > wrong "open" ↓ generator.py → Claude-powered cover letters │ Tone mapping: startup/consulting/research │ Banned hollow phrases (89% of AI CLs fail this) │ Problem-Solution format (49% more interviews) ↓ exporter.py → 4-sheet styled Excel export ↓ Next.js 14 frontend → Google OAuth + Stripe + Dashboard

What Makes It Different

Ghost Job Killer

Every posting verified against real ATS APIs. "Unverified" when ambiguous — never lies about open status. Daily re-verification daemon purges stale jobs.

Profile-Aware Search

Queries built from YOUR resume, not generic templates. Degree filtering, career stage separation (intern vs new grad), and category-tagged company matching.

Human Cover Letters

Claude generates letters that sound like a person, not a template. Bans 20+ hollow phrases ("passionate about", "fast learner"). Tone adapts to company culture.

Production SaaS

Google OAuth, Stripe billing ($12/mo Pro), rate limiting, PostgreSQL, Railway backend, Vercel frontend. Real users, real payments.

Key Technical Decisions

  • Verification philosophy: "An honest unverified beats a wrong open." ATS API first (definitive), HTML signals second, ghost detection by posting age
  • Company registry: 50+ curated companies with category tags (DS/ML, Health Informatics, SWE, PM) — not scraping LinkedIn
  • Cover letter quality: Based on analysis of 80+ studies — Problem-Solution format, 49% more interviews, 72% of recruiters prioritize customization
  • Daily refresh daemon: Re-verifies all cached jobs automatically, removes closed postings before users see them

Tech Stack

Next.js 14, TypeScript, Tailwind CSS, Flask, SQLAlchemy, PostgreSQL, Claude API (Anthropic), Stripe, Google OAuth (NextAuth), Railway, Vercel, PyMuPDF
